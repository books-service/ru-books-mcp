# Политика безопасности

Если вы обнаружили проблему безопасности, пожалуйста, не публикуйте ее сразу публично.

## Как сообщить о проблеме

- Откройте приватный отчет через security advisories репозитория (предпочтительно), или
- Свяжитесь с сопровождающими напрямую и приложите:
  - описание проблемы,
  - потенциальное влияние,
  - шаги воспроизведения,
  - предлагаемый способ устранения (если есть).

## Что относится к security-рискам

К проблемам безопасности относятся (в том числе):
- обход аутентификации,
- утечка данных,
- риски request smuggling/десериализации,
- векторы отказа в обслуживании (DoS), специфичные для MCP endpoint'ов.
