# Участие в проекте

Спасибо за интерес к RU Books MCP.

## Как внести вклад

1. Откройте issue с четким описанием проблемы.
2. Предлагайте сфокусированные изменения (одна задача на PR).
3. Для изменений поведения добавляйте воспроизводимые шаги проверки.
4. Сохраняйте обратную совместимость контрактов инструментов, если явно не вводится новая версия.

## Чеклист pull request

- [ ] Понятное описание изменений
- [ ] Обновлена документация для пользовательского поведения
- [ ] Нет секретов/токенов в коде и примерах
- [ ] Пройдена базовая проверка endpoint'ов (`/health`, `/mcp initialize`)
